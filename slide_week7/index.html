Local Minimum vs Global Minimum

Local Minimum

Thấp nhất trong vùng lân cận

Như cái hố nhỏ - đứng trong hố thì thấy mình thấp nhất, nhưng có hố sâu hơn ở chỗ khác

Global Minimum

Thấp nhất toàn bộ

Cái hố sâu nhất - đây mới là đáp án thật sự mà ta muốn tìm

Vấn đề lớn: Gradient Descent chỉ "nhìn" xung quanh → dễ kẹt ở local min. GA và PSO thử nhiều vị trí cùng lúc → thoát được local min tốt hơn.

Gradient Descent

Ý tưởng: Bi lăn xuống dốc

Đặt viên bi ở 1 vị trí bất kỳ trên đồ thị → nó tự lăn xuống chỗ thấp nhất.
Dốc = đạo hàm. Dốc nhiều → bước lớn. Gần đáy → bước nhỏ dần.
xmới = xcũ − lr × f'(xcũ)

xcũ

Vị trí hiện tại
của viên bi

lr (learning rate)

Kích thước bước đi
Chọn bao nhiêu? → slide sau

f'(x) (đạo hàm)

Độ dốc tại vị trí hiện tại
Âm → đi sang phải, Dương → đi sang trái

Learning Rate - Kích thước bước đi

lr quá nhỏ (0.001)

Bước đi rất nhỏ
Chậm rì → tốn hàng nghìn bước
Có thể chưa đến đáy đã hết kiên nhẫn

lr vừa phải (0.1)

Bước đi vừa đủ
Hội tụ nhanh và chính xác
Đây là mục tiêu!

lr quá lớn (1.5)

Bước đi quá lớn
Nhảy qua nhảy lại, không hội tụ
Có thể bay ra vô cực!

Quy tắc thực tế: Thử lr = 0.01, 0.1, 0.5. Nếu dao động → giảm lr. Nếu quá chậm → tăng lr.

GD 1 biến: Code tay

Tìm min của f(x) = x² − 4x + 5 → đáp án chính xác: x = 2, f(2) = 1

def f(x): return x**2 - 4*x + 5 def df(x): # đạo hàm f'(x) = 2x - 4 return 2*x - 4 # Gradient Descent x = 0.0 # xuất phát từ x = 0 lr = 0.1 # learning rate for i in range(20): grad = df(x) # tính đạo hàm tại x x = x - lr * grad # cập nhật x print(f"Bước {i+1}: x = {x:.4f}, f(x) = {f(x):.4f}") # Kết quả: x → 2.0000, f(x) → 1.0000
Chỉ 6 dòng code chính! Vòng lặp: tính đạo hàm → cập nhật x → lặp lại. Đơn giản vậy thôi.

GD 2 biến: Contour Plot

Tìm min của f(x, y) = x² + y² → đáp án: (0, 0), f = 0

# Gradient: ∇f = (∂f/∂x, ∂f/∂y) = (2x, 2y) x, y = 3.0, 4.0 # xuất phát lr = 0.1 for i in range(30): dx = 2 * x # ∂f/∂x dy = 2 * y # ∂f/∂y x = x - lr * dx # cập nhật x y = y - lr * dy # cập nhật y print(f"Kết quả: ({x:.4f}, {y:.4f})") # → (0.0000, 0.0000)
Mở rộng tự nhiên: 1 biến → đạo hàm f'(x). 2 biến → gradient ∇f = (∂f/∂x, ∂f/∂y). Cùng công thức, chỉ thêm chiều.

from scipy.optimize import minimize # Cách 1: Hàm 1 biến f(x) = x² - 4x + 5 ket_qua = minimize(lambda x: x[0]**2 - 4*x[0] + 5, x0=[0]) print(ket_qua.x) # [2.0] print(ket_qua.fun) # 1.0 # Cách 2: Hàm 2 biến f(x,y) = x² + y² ket_qua = minimize(lambda p: p[0]**2 + p[1]**2, x0=[3, 4]) print(ket_qua.x) # [0.0, 0.0] print(ket_qua.fun) # 0.0

Genetic Algorithm

Hạn chế của Gradient Descent

Vấn đề 1: Kẹt Local Min

Hàm có nhiều "hố" → GD rơi vào hố gần nhất, không phải hố sâu nhất.

VD: f(x) = x⁴ − 8x² + x
Xuất phát bên phải → kẹt ở hố phải, bỏ lỡ hố trái sâu hơn

Vấn đề 2: Cần đạo hàm

GD bắt buộc phải tính f'(x). Nhưng nhiều bài không có đạo hàm:

  • Xếp lịch thi (biến rời rạc: slot 1, 2, 3...)
  • Tuyến giao hàng (biến là hoán vị)
  • Chọn nguyên liệu (có/không)
Giải pháp: Dùng thuật toán không cần đạo hàm, lấy cảm hứng từ tự nhiên!
→ Genetic Algorithm (GA): học từ tiến hóa sinh học
→ Particle Swarm Optimization (PSO): học từ đàn chim/cá

GA: Ý tưởng từ tiến hóa

Trong tự nhiên: cá thể mạnh sống sót, sinh sản, tạo thế hệ sau tốt hơn.
GA mô phỏng quá trình này để tìm lời giải tối ưu.

Sinh học

Cá thể = 1 con vật
Gene = DNA
Fitness = khả năng sinh tồn

GA

Cá thể = 1 lời giải
Gene = mảng số
Fitness = f(x) tốt cỡ nào

VD: Giao hàng

Cá thể = [3,1,5,2,4]
Gene = thứ tự ghé khách
Fitness = tổng quãng đường

GA: 4 bước hoạt động

Lặp lại 100-1000 thế hệ. Mỗi thế hệ, quần thể tốt dần lên vì cá thể yếu bị loại, cá thể mạnh được giữ lại và tạo con.

GA: Code tay

Tìm min f(x) = x⁴ − 8x² + x trên [-5, 5] (hàm có nhiều local min)

import numpy as np def f(x): return x**4 - 8*x**2 + x # 1. Khởi tạo quần thể 50 cá thể ngẫu nhiên pop = np.random.uniform(-5, 5, size=50) for gen in range(100): fitness = f(pop) # Tính fitness # 2. Chọn lọc: giữ 20 cá thể tốt nhất (f nhỏ nhất) best_idx = np.argsort(fitness)[:20] parents = pop[best_idx] # 3. Lai ghép: trung bình 2 cha mẹ ngẫu nhiên children = [] for _ in range(30): p1, p2 = parents[np.random.choice(20, 2, replace=False)] child = (p1 + p2) / 2 # Lai ghép đơn giản children.append(child) # 4. Đột biến: thêm nhiễu nhỏ children = np.array(children) children += np.random.normal(0, 0.3, 30) pop = np.concatenate([parents, children]) # Thế hệ mới print(f"x tối ưu = {pop[np.argmin(f(pop))]:.4f}") print(f"f(x) = {f(pop[np.argmin(f(pop))]):.4f}")

GA: Thư viện DEAP

Cùng bài toán, dùng thư viện chuyên nghiệp

from deap import base, creator, tools, algorithms import numpy as np def f(individual): x = individual[0] return (x**4 - 8*x**2 + x,) # DEAP yêu cầu tuple # Cấu hình creator.create("FitnessMin", base.Fitness, weights=(-1.0,)) creator.create("Individual", list, fitness=creator.FitnessMin) toolbox = base.Toolbox() toolbox.register("attr", np.random.uniform, -5, 5) toolbox.register("individual", tools.initRepeat, creator.Individual, toolbox.attr, n=1) toolbox.register("population", tools.initRepeat, list, toolbox.individual) toolbox.register("evaluate", f) toolbox.register("mate", tools.cxBlend, alpha=0.5) toolbox.register("mutate", tools.mutGaussian, mu=0, sigma=0.3, indpb=1.0) toolbox.register("select", tools.selTournament, tournsize=3) # Chạy GA pop = toolbox.population(n=50) result = algorithms.eaSimple(pop, toolbox, cxpb=0.7, mutpb=0.2, ngen=100) best = tools.selBest(pop, 1)[0] print(f"x = {best[0]:.4f}, f(x) = {f(best)[0]:.4f}")

Particle Swarm Optimization

PSO: Ý tưởng từ đàn chim

Đàn chim tìm thức ăn: mỗi con bay theo hướng riêng, nhưng luôn học từ:
1) Chỗ tốt nhất mình từng đến    2) Chỗ tốt nhất cả đàn từng đến

Particle (hạt)

= 1 lời giải
Có vị trí (x) và vận tốc (v)
Di chuyển trong không gian tìm kiếm

pBest (cá nhân)

Vị trí tốt nhất mà
bản thân particle từng đến
"Kinh nghiệm cá nhân"

gBest (toàn cục)

Vị trí tốt nhất mà
cả đàn từng đến
"Trí tuệ tập thể"

PSO: Công thức cập nhật

Vận tốc mới:
v = w × v + c₁ × rand() × (pBest − x) + c₂ × rand() × (gBest − x)
Vị trí mới:   x = x + v

w × v

Quán tính
Tiếp tục bay theo hướng cũ
w = 0.5 ~ 0.9

c₁ × (pBest − x)

Học từ bản thân
Kéo về chỗ tốt nhất mình từng đến
c₁ = 1.5 ~ 2.0

c₂ × (gBest − x)

Học từ đàn
Kéo về chỗ tốt nhất cả đàn
c₂ = 1.5 ~ 2.0

PSO: Code tay

Cùng bài: tìm min f(x) = x⁴ − 8x² + x trên [-5, 5]

import numpy as np def f(x): return x**4 - 8*x**2 + x n = 30 # 30 particles x = np.random.uniform(-5, 5, n) # vị trí ban đầu v = np.random.uniform(-1, 1, n) # vận tốc ban đầu pbest = x.copy() # best cá nhân gbest = x[np.argmin(f(x))] # best toàn cục w, c1, c2 = 0.7, 1.5, 1.5 for _ in range(100): r1 = np.random.rand(n) r2 = np.random.rand(n) v = w*v + c1*r1*(pbest - x) + c2*r2*(gbest - x) # cập nhật vận tốc x = x + v # cập nhật vị trí # Cập nhật pBest và gBest better = f(x) < f(pbest) pbest[better] = x[better] gbest = pbest[np.argmin(f(pbest))] print(f"x tối ưu = {gbest:.4f}, f(x) = {f(gbest):.4f}")
Chỉ 3 dòng quan trọng: Cập nhật v → cập nhật x → cập nhật pBest/gBest. Lặp lại 100 vòng.

PSO: Thư viện pyswarm

Cùng bài toán, chỉ cần 4 dòng

from pyswarm import pso def f(x): return x[0]**4 - 8*x[0]**2 + x[0] lb = [-5] # lower bound ub = [5] # upper bound x_opt, f_opt = pso(f, lb, ub, swarmsize=30, maxiter=100) print(f"x = {x_opt[0]:.4f}, f(x) = {f_opt:.4f}")

# Bài 2 biến: f(x,y) = x² + y² + noise (nhiều local min) def f2(p): x, y = p return x**2 + y**2 + 2*np.sin(5*x) + 2*np.sin(5*y) x_opt, f_opt = pso(f2, [-5,-5], [5,5], swarmsize=50, maxiter=200) print(f"(x,y) = ({x_opt[0]:.3f}, {x_opt[1]:.3f}), f = {f_opt:.4f}")

• Hàm liên tục, có đạo hàm → GD (nhanh nhất)
